# 夏夜将终 OFFICIAL SITE — 内容贡献指南

本文件面向两类人：

- **不太懂代码的乐队成员**：照着"快速上手"做即可，不需要改 HTML / JS。
- **AI 或开发者**：照着"给 AI / 开发者的字段规范"做批量更新。

> 一句话总览：这个网站的几乎所有动态内容（新闻、演出、作品、MV、成员、年表、联系方式）都集中在一个文件——`config.json`。**图片/音频/视频文件先放进 `assets/img/`（或新建目录），再把对应的文字信息和 B 站 BV 号写进 `config.json`，刷新页面即生效。** 不需要改任何 HTML 或 JS。

---

## 1. 项目是怎么运作的（先理解这一点）

```
config.json   ← 唯一的"内容数据库"，所有文字 / 链接 / 图片路径都在这里
   │
   ▼
render.js      ← 打开页面时读取 config.json，把内容"填"进各页面
   │
   ▼
index.html / news.html / live.html / ...   ← 只负责提供页面骨架和样式，本身没有内容
```

- 页面骨架（HTML）和样式（CSS）是从 Yorushika 官网复刻来的，**几乎不需要动**。
- 所有"内容"都靠 `config.json` 动态注入。改 `config.json` → 刷新 → 生效。
- 本地预览：在项目根目录运行 `python -m http.server 8080`，再浏览器打开 http://localhost:8080/（直接双击 HTML 打开会因为跨域限制加载不了 config.json，必须用本地服务器）。

## 2. 目录约定（素材往哪放）

| 素材类型 | 放置位置 | 说明 |
|---|---|---|
| 作品封面 / 头图 / banner | `assets/img/banner/` | 当前命名：`v01_xxx.jpg`、`bnr01_xxx.jpeg` |
| 顶部主视觉 / 背景 | `assets/img/top/` | `kv.jpg`、`bg.jpg` |
| Logo | `assets/img/logo/` | `handwriting.png` 等 |
| 页脚横幅 | `assets/img/footer/` | 友链三张图 |
| 通用占位 | `assets/img/common/` | `dummy.png` |
| 站点头像 / favicon | `assets/img/avatar.jpg`、`assets/img/favicon.ico`、`assets/img/ogimg.png` | |
| 自定义静态页（如即兴复活 EP02 海报页） | `feature/` | 例如 `feature/livetour2026_ichininsho.html` |

> 当前**没有**专门的音频/视频目录：音频和视频都以 **B 站 BV 号**的形式引用（在 `config.json` 里填 `bvid`），网站会自动生成跳转链接和封面。若以后要托管本地音频/视频，建议新建 `assets/media/` 放置。

**图片命名建议**：用全小写、英文或拼音、带序号，避免空格和中文，例如 `v08_25_march_boiling.jpeg`。文件路径在 `config.json` 里要写成**相对站点根目录的路径**（如 `assets/img/banner/v08_xxx.jpg`，开头不要加 `/`，也不要写 `./`），这样在本地和 GitHub Pages 上都能正常显示。

## 3. 快速上手：我要加一条新内容，怎么做？

### 3.1 加一条新闻（NEWS）

1. 准备好 B 站视频的 BV 号（例如 `BV1yTZmYdEeX`）。
2. 打开 `config.json`，找到 `"news"` 数组，在**最前面**加一行（新闻按时间倒序显示，新的放前面）：

```json
{ "date": "2025.05.01", "category": "翻唱", "title": "新翻唱标题", "bvid": "BVxxxxxxxx" }
```

3. 如果这条新闻是对应一个翻唱/作品，还要同时在 `discography` 和 `movies` 数组里加上同一条（见 3.3），否则首页"作品轮播图"和 MOVIE 列表里看不到。

`category` 目前用过的值：`翻唱`、`LIVE`、`活动`。可以自定义，但建议沿用现有值，保持一致。

### 3.2 加一场演出日程（LIVE）

在 `config.json` 的 `"live"` 数组加：

```json
{ "date": "2025.06.01", "week": "SAT", "title": "演出名称@场地", "bvid": "" }
```

- `week`：星期几的英文缩写，如 `SUN`、`SAT`。
- `bvid`：如果有对应的 B 站回顾视频就填，没有就留空 `""`。
- LIVE 页面会把 `title` 里 `@` 后面的当作场地（仅文字展示，不强制格式）。

> 注意：`live` 数组目前只用来排"未来/近期日程"。已结束演出的归档目前主要写在 `biography.history` 里。详见第 5 节"目前的一个小坑"。

### 3.3 加一个翻唱作品 / 视频（DISCOGRAPHY + MOVIE）

一个作品要**同时**改三个地方，因为它会出现在"首页作品轮播""DISCOGRAPHY 列表""MOVIE 列表"三处：

1. 把封面图放进 `assets/img/banner/`，记住文件名。
2. 在 `"discography"` 数组最前面加：

```json
{
  "title": "曲名",
  "date": "2025.06.01",
  "category": "翻唱",
  "original": "原唱 / 原作者",
  "cover": "assets/img/banner/v08_xxx.jpg",
  "bvid": "BVxxxxxxxx"
}
```

3. 在 `"movies"` 数组最前面加（标题可短一些，不需要 cover——会自动从 discography 里按 bvid 匹配封面）：

```json
{ "title": "曲名-ヨルシカ", "bvid": "BVxxxxxxxx", "date": "2025.06.01" }
```

4. （可选）在 `"news"` 数组也加一条公告，让首页 NEWS 也显示。

`category` 用过的值：`翻唱`、`活动`。LYRICS 页面只显示 `category === "翻唱"` 的作品。

### 3.4 改乐队简介 / 成员 / 年表（BIOGRAPHY）

全部在 `config.json` 的 `"biography"` 对象里：

- `photo`：BIOGRAPHY 页那张大图的路径。
- `intro`：乐队简介文字，`\n` 表示换行。
- `members`：成员数组，`role`（如 `Vocal`）+ `name`。
- `history`：年表数组，`year`（如 `2025年`）+ `text`，`text` 里 `\n` 换行。

### 3.5 改联系方式 / 友链 / 站点配色

- `"contact"` 和 `"social"`：B 站主页链接和账号名。
- `"footerBanners"`：页脚三张友链横幅图 + 链接。
- `"site"`：站点名称、配色（`color` / `colorAccent` / `colorTeal`）。

### 3.6 加歌词（LYRICS）

⚠️ 目前**不支持**通过 `config.json` 直接填歌词正文。`lyrics-detail.html` 现在写死显示"歌詞は準備中です"占位（见 `assets/js/detail-render.js` 的 `renderLyricsDetail`）。要加歌词需要改 JS，属于开发工作。详见 reports 里的代码问题报告。

## 4. 给 AI / 开发者的字段规范

### config.json 顶层结构

```jsonc
{
  "site": { "name", "nameJP", "nameCN", "description", "color", "colorAccent", "colorTeal" },
  "social": { "bilibili", "bilibiliName" },
  "footerBanners": [ { "img", "alt", "link", "target" } ],   // 固定 3 张
  "biography": { "photo", "intro", "members":[{role,name}], "history":[{year,text}] },
  "news": [ { "date":"YYYY.MM.DD", "category", "title", "bvid" } ],          // 倒序，首页取前 3
  "live": [ { "date":"YYYY.MM.DD", "week", "title", "bvid" } ],
  "discography": [ { "title", "date", "category", "original", "cover", "bvid" } ],
  "movies": [ { "title", "bvid", "date" } ],                // 封面按 bvid 从 discography 取
  "works": [ /* 同 news 形式: {date,category,title} */ ],     // 目前为空
  "contact": { "bilibili", "bilibiliName" }
}
```

### 日期格式

统一 `YYYY.MM.DD`，如 `2025.04.03`。`week` 用三字母英文星期（`MON`/`TUE`/`WED`/`THU`/`FRI`/`SAT`/`SUN`）。

### 路径格式

`cover` / `photo` / `img` 等一律写 **相对站点根**，例如 `assets/img/banner/v08_xxx.jpg`。不要写 `./`、不要写绝对盘符路径、不要写 `http`（除非引用外部图，不建议）。

### 添加一条"完整作品"的标准动作清单（AI 执行时请一次做完）

1. 把封面图落到 `assets/img/banner/`（如由用户提供二进制，AI 无法直接上传二进制时，提示用户先放好并告知文件名）。
2. 取得 B 站 BV 号。
3. 在 `discography` 头部插入对象（含 `cover`、`original`、`bvid`）。
4. 在 `movies` 头部插入对象（含 `bvid`）。
5. 在 `news` 头部插入公告对象（含 `bvid`）。
6. 若是演出而非翻唱，根据情况进 `live` 和/或 `history`，并设 `category:"活动"` 或 `"LIVE"`。
7. 校验 JSON 合法性（`python -m json.tool config.json`）。
8. 刷新页面核对三处（首页作品轮播 / DISCOGRAPHY / MOVIE / NEWS）。

### JSON 校验小命令

```powershell
python -m json.tool config.json
```

无报错即合法（注意 config.json 带有 UTF-8 BOM，部分旧版本工具可能报 BOM 警告，不影响浏览器加载）。

## 5. 目前的一个小坑（务必注意）

- **"完整作品"必须同时出现在 `discography` + `movies`（+ 可选 `news`）三个数组**，否则有的页面看不到。这是当前设计的重复，AI/人工添加时容易漏掉 `movies`。
- `live` 数组目前只有"即兴复活 EP02"一条，**已结束的演出没有统一的归档入口**，只能写在 `biography.history` 的年表文字里。如果想做"过往演出回顾列表"，目前没有现成字段，需要扩展（见代码问题报告）。
- LYRICS 歌词正文无法靠 config.json 添加，需改 `detail-render.js`。
- 直接双击 HTML 打开页面会空白（config.json 加载失败），必须用本地服务器 `python -m http.server`。
- 部分页面（如 `special.html`、`works.html`）是"凑数"的占位实现：SPECIAL 页面其实是把 discography 前 4 个封面再展示一遍，WORKS 页面则是"暂无信息"占位。

## 6. 给非技术成员的"最简流程"小结

你要做的只有两件事：

1. **图片**：把图片文件放进 `assets/img/banner/` 文件夹（让懂的人帮你，或者直接拖进去），记住文件名。
2. **文字**：告诉 AI（或会改 JSON 的人）日期、标题、原唱、B 站链接，让它帮你写进 `config.json`。

如果你会改 JSON：打开 `config.json`，照着已有条目的样子复制一行改几个字就行。改完保存，刷新浏览器。不确定就用 `python -m json.tool config.json` 验证一下。
